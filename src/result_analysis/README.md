# Result Analysis (`src/result_analysis/`)

The Result Analysis module standardizes the evaluation, reporting, and visualization of simulated strategy outcomes generated by the `backtest_engine`. It provides a metrics library, a one-call `BacktestReport` class, and a plotters library for comprehensive performance tearsheets.

> **IMPORTANT**: All backtests — both manual and Qlib-based — **must** use this module for performance analysis. Do NOT write custom metric functions in notebooks or scripts. If a metric is missing, add it here.

## Architecture & Responsibilities

```text
result_analysis/
├── __init__.py     # Module initialization and convenience re-exports
├── metrics.py      # Pure functions for performance KPIs + 果仁-style trading stats
├── report.py       # BacktestReport class — one-call analysis with styled tables
└── plotters.py     # Visualization functions (Plotly interactive + matplotlib fallback)
```

---

## Public API Reference

### `BacktestReport` (`report.py`) — One-Call Analysis Class

The primary entry point for backtest evaluation. Combines metrics + plots in a single class.

```python
from src.result_analysis import BacktestReport

report = BacktestReport(strategy_returns, benchmark_returns, name="My Strategy")
report.summary()         # Metrics table (styled in Jupyter)
report.plot()            # Interactive Plotly dashboard
report.yearly()          # Yearly breakdown table
report.monthly_heatmap() # Month×Year heatmap
report.rolling()         # Rolling Sharpe/Vol/Return
report.distribution()    # Return histogram
report.trading_analysis(holdings, df)  # 果仁-style trading stats
report.to_html("out.html")
```

| Method | Description |
|--------|-------------|
| `summary()` | Styled metrics table: CAGR, Sharpe, Sortino, MDD, Calmar, Win Rate, etc. |
| `plot(**kwargs)` | Interactive Plotly dashboard (equity, drawdown, monthly heatmap) |
| `yearly()` | Per-year returns, volatility, Sharpe, MDD breakdown |
| `monthly_heatmap()` | Month × Year return heatmap |
| `rolling(window=60)` | Rolling Sharpe, volatility, and return charts |
| `distribution()` | Daily return histogram with KDE |
| `trading_analysis(holdings, df, ...)` | 果仁-style trading statistics (see below) |
| `to_html(path)` | Export full report as standalone HTML |

| `calculate_max_drawdown` | Maximum peak-to-trough decline |
| `calculate_max_drawdown_duration` | Longest drawdown recovery in days |
| `calculate_drawdown_series` | Full drawdown time series |
| `calculate_sharpe_ratio` | Annualized Sharpe ratio |
| `calculate_sortino_ratio` | Downside-only risk-adjusted return |
| `calculate_calmar_ratio` | CAGR / Max Drawdown |
| `calculate_information_ratio` | Excess return / tracking error |
| `calculate_alpha_beta` | CAPM alpha and beta vs benchmark |
| `calculate_win_rate` | Fraction of positive-return days (日赢率) |
| `calculate_weekly_win_rate` | Fraction of positive-return weeks (周赢率) |
| `calculate_monthly_win_rate` | Fraction of positive-return months (月赢率) |
| `calculate_profit_factor` | Sum of gains / sum of losses |
| `calculate_tail_ratio` | 95th percentile / abs(5th percentile) |
| `calculate_skewness` | Return distribution skew |
| `calculate_kurtosis` | Return distribution kurtosis |
| `calculate_tracking_error` | Annualized std(excess return) vs benchmark |

### `generate_trading_stats()` — 果仁-Style Trading Analysis

Computes 15 position-level trading metrics matching the 果仁 (Guoren) backtesting platform's output format. This function requires position-level data, not just returns.

```python
from src.result_analysis.metrics import generate_trading_stats

stats = generate_trading_stats(
    holdings=holdings,         # Dict {pd.Timestamp: [stock_codes]}
    df=df,                     # Full DataFrame with adj_close, vol, daily_ret, is_limit_up/down
    strategy_returns=rets,     # Daily return series (for win rates)
    benchmark_returns=bench,   # Benchmark returns (for tracking error)
    report_df=qlib_report,     # Qlib report DataFrame (for turnover)
    buy_cost=0.0005,
    sell_cost=0.0015,
)
```

**Output metrics:**

| Metric | Chinese | Description |
|--------|---------|-------------|
| Avg Stocks Held | 平均持仓股票数 | Average number of stocks per rebalance |
| Stock Changes | 换股次数 | Total buy+sell transactions across all rebalances |
| Avg Holding Days | 平均持有天数 | Mean calendar days per completed trade |
| Avg Trade Return | 平均交易收益 | Mean return per completed trade (net of costs) |
| Avg Winning Trade | 正收益平均 | Mean return of profitable trades |
| Avg Losing Trade | 负收益平均 | Mean return of losing trades |
| Trade Win Rate | 交易赢率 | Fraction of trades with positive return |
| Annual Turnover | 年换手率 | Annualized portfolio turnover |
| Suspended Ratio | 持仓停牌股票比例 | Fraction of held stock-days with zero volume |
| Daily Win Rate | 日赢率 | Fraction of positive-return days |
| Weekly Win Rate | 周赢率 | Fraction of positive-return weeks |
| Monthly Win Rate | 月赢率 | Fraction of positive-return months |
| Tracking Error | 指数跟踪误差 | Annualized std of excess returns |
| Order Exec Rate | 调仓指令可执行比例 | Fraction of rebalance orders not blocked by limits |
| Avg Position | 平均持仓仓位 | Average capital utilization |

### `plotters.py` — Visualization Library

Interactive Plotly charts with matplotlib fallback via `plot_comprehensive_report_mpl()`.

| Function | Description |
|----------|-------------|
| `plot_equity_curve(...)` | Cumulative return line chart (Plotly) |
| `plot_drawdown(...)` | Underwater chart with red fill (Plotly) |
| `plot_excess_return(...)` | Strategy minus benchmark cumulative return |
| `plot_monthly_heatmap(...)` | Month × Year heatmap of returns |
| `plot_yearly_returns(...)` | Bar chart of annual returns |
| `plot_return_distribution(...)` | Histogram + KDE of daily returns |
| `plot_rolling_metrics(...)` | Rolling Sharpe, vol, return (Plotly) |
| `plot_dashboard(...)` | Multi-panel Plotly dashboard |
| `plot_comprehensive_report_mpl(...)` | Matplotlib 3-panel tearsheet (for PDF export) |

## Usage

```python
from src.result_analysis import BacktestReport, generate_trading_stats

# ─── Option 1: One-call BacktestReport ──────────────────
report = BacktestReport(strategy_returns, benchmark_returns, name="Small-Cap")
report.summary()
report.plot()
report.trading_analysis(holdings, df, buy_cost=0.0005, sell_cost=0.0015)

# ─── Option 2: Standalone metrics ───────────────────────
from src.result_analysis.metrics import generate_performance_report
perf = generate_performance_report(strategy_returns, benchmark_returns)
print(perf)

# ─── Option 3: Standalone trading stats ─────────────────
stats = generate_trading_stats(holdings, df, strategy_returns=rets, benchmark_returns=bench)
print(stats)
```
