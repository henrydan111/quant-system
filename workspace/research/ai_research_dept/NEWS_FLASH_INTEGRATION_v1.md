## §2 噪音去除管线(设计核心;确定性优先、LLM 其次、全部 fail-closed)

按序执行,每级淘汰都留结构化计数(进事件库审计字段,绝不静默丢弃):

1. **源分层白名单**:v1 仅收实测可用的 4-5 源;`source_tier` 分层(cls/wallstreetcn=T1,
   sina/eastmoney/10jqka=T2)。新源加入 = 配置版本变更。
2. **确定性预过滤**(零 LLM 成本,先砍量):长度域(<20 字弃);channels 栏目黑名单
   (娱乐/体育等);**黑嘴模式词表硬丢弃**(荐股话术:目标价/建仓/加群/涨停板战法等
   ——反操纵第一道,词表版本化)。
3. **跨源去重 + 佐证计数**:标题/正文规范化指纹聚簇(实测 18% 粗重复);簇内保留最早
   `visible_at` 为主事件,`n_sources` 记佐证度——**把重复从噪音变成信号**(多源同报
   = 重要度加分项,进分型输入)。
4. **实体挂钩三路路由器**(v1.1:从「过滤门」改为「路由器」——宏观流是数据资产,
   不是垃圾):
   - **个股路**:代码/股票名/别名(含 H 股/ADR 别名表)精确匹配 → `subject_codes`
     → news 卡直接事件节(NF 行);
   - **行业/概念路**:申万行业词表 + THS 概念词表命中 → `industry_tags/concept_tags`
     → news 卡间接事件节(NDA 聚合/钳位照旧);
   - **宏观路**(v1.1 新增):货币/财政/地缘/大盘异动/外围市场/商品汇率/监管全局类
     → `macro` 通道 → **宏观卡**(§6),供第四席消费;
   - **丢弃**仅限:预过滤已拦的黑嘴/娱乐类 + 三路全不命中且分型为「市场评论」的
     无信息闲聊(逐级计数留审计)。
5. **quick-LLM 分型**(复用 irm_qa 分型基建:doubao-lite thinking OFF、注册 enum、
   校验 fail-closed):个股/行业路 `event_type ∈ {公司经营, 订单/合同, 产能/产品,
   行业动态, 政策转述, 盘面异动, 传闻/未证实, 市场评论}`;宏观路(v1.1)
   `macro_type ∈ {货币政策, 财政政策, 监管全局, 地缘/外围, 大盘/资金面, 商品/汇率,
   行业景气}`;两路共用 `direction`/`importance_0_5`;
   **`is_rumor` 旗:单源 + 未证实表述 → importance 封顶 2**。
6. **新颖度门**:与近 3 日同主体事件指纹比对,重复报道降权(novelty 字段),
   防同一故事连日刷屏。
7. **卡片端机械围栏**(消费侧,复用既有 validators 基建):新行 ID 类 **NF##**;
   `_news_requires_cap` 加 NF 前缀 → **NF 类证据对分数的贡献钳位 ≤3**(与 NI/NDA 同法);
   同源同类配额;间接快讯走 NDA 聚合;缺席谓词照旧(「窗口内无池内快讯」显式声明)。
8. **反操纵评分护栏**(KOL/情绪深研裁决落地):**`传闻/未证实` 类 NF 行不得作为
   score≥4 维度的唯一证据**(scorecard 端 validator 规则,机械执行);传闻旗事件
   自动进空头席证伪素材。

## §3 PIT 锚定

- `visible_at = datetime`(发布秒级);**前向抓取**加 `fetch_ts`,生效锚 =
  `max(datetime, fetch_ts)`(防迟到/回填行伪装历史可见——B1/B2 同款防御,快讯爬虫流
  风险低但审计字段必须在)。
- 历史重放本就 NON_EVIDENTIARY;快讯历史深度 6 年+,是否存在编辑/回填未经验证——
  入库记 `fetch_ts`,历史批次统一标 `history_bulk`,不承诺行级 PIT。

## §6 宏观市场分析师(第四席,v1.1 用户裁定)

**角色定位:** 消费宏观/市场数据资产,并把宏观情境**传导为逐股指引**——不是给大盘
打分,而是回答「这只股票在当前宏观情境下顺风还是逆风」。

**输入 = 宏观卡(新卡,当日全池共享)+ 该股身份标签:**
- 市场情境卡 v0.4 全文(M01-M16 行,既有);
- **宏观快讯节**(v1.1 新增):宏观路 NF 按 macro_type 分组,重要度排序取 top-N
  (配额,如每类 ≤3/合计 ≤12),带佐证数与事件龄,行 ID 新类 **MF##**;
- 政策事件节(npr/联播/货政,既有事件);
- 该股的申万行业 + THS 概念标签 + 风格标记(市值分层/β,来自 fact/pv 既有字段)。

**输出 = 宏观传导 scorecard**(与既有席位同构,全部走既有 C16 围栏:注册维恰一次、
0-5 分、逐字 evidence_spans 接地到 M##/MF## 行、两遍独占、缺席谓词):
维度(五维,权重结构与既有席同法 Σw×5=100,权重进 scoring_contract 冻结):
1. 风险偏好环境适配(当前 regime 对该股风格的顺逆);
2. 流动性/资金面传导(两融/北向/大盘量能环境对该股筹码结构的含义);
3. 行业/概念景气传导(宏观快讯 → 该股行业/概念的映射,必须引用 MF 行);
4. 政策暴露(方向带符号:受益/受损/无关);
5. 情绪位置(该股所处板块的拥挤度/逆向风险)。

**接线:** run_seat 四席并列(macro 席 payload = 宏观卡 + 股票标签摘要,prompt 新文件
`macro_analyst_v1.txt` 进 manifest 冻结);空头席照常消费四席 scorecard;judge
composite 改四席权重(**建议 fund 0.35 / tech 0.25 / news 0.20 / macro 0.20,
预注册参数,待用户/GPT 裁定**);背离旗两两比较自然覆盖第四席。

**成本:** +1 调用/股(宏观卡全池共享 → prompt 缓存命中率高,实测 cached_tokens
已生效);单股 ~9.5 → **~12 AFP**;分层 4 日验证 ≈ 7.5k AFP,仍远低于取消的全月。

**为什么是逐股席而不是每日一次+机械映射:** 传导判断(「美联储鹰派 → 高β成长股
逆风」vs「地产政策 → 该股上游敞口」)需要股票上下文,机械映射表达不了;每日一次的
全池共享宏观卡 + 逐股传导调用,是表达力与成本的平衡点(缓存把重复上下文的成本
压掉)。备选方案(每日一次宏观简报 + 确定性行业映射)作为 GPT 评审的对照项。

## §5 实现清单(GPT 审后)

1. `TushareFetcher.fetch_news`(串行/时间窗循环/1500 分页)+ 月分片 parquet
   (`data/analyst/news_flash/`)

   schema,source=`news_flash`,payload 带 n_sources/novelty/is_rumor/tier/route)
3. 检索层 `news_flash` 通道 + 配额;news 卡 NF 行渲染(NF ID 注册表);
   **宏观卡 builder + MF ID 注册表**(§6)
4. **宏观席**:`macro_analyst_v1.txt` prompt + SEAT_WEIGHTS/COMPOSITE_W 四席化
   (进 scoring_contract)+ judge/bear/archive/平台四席适配
5. validators:NF/MF 钳位 + 传闻不撑高分规则;prompts 增消费说明
6. 测试:管线逐级淘汰计数/三路路由/去重簇/分型 enum fail-closed/NF·MF 钳位/
   传闻护栏/四席 composite 重算一致性(verify_archive_semantics 键集自动随契约)
7. 链版本 bump(v3.1+)→ 单日烟测 → 分层 3-4 日验证(替代全月)
